How your password is derived

Every password is produced by feeding three pieces of information into Argon2id, a memory-hard key derivation function. Because all steps are deterministic, entering the same inputs always produces the same output — yet nothing needs to be stored anywhere.

Step 1 — Build the salt

The Username, Domain, and optional Salt Pepper are joined into a single string username:domain:pepper (username and domain are forced lowercase, pepper as entered). That string is hashed with SHA-256, and the first 16 bytes of the digest are used as the Argon2id salt. This ties the salt uniquely to the service without requiring you to store it.

Step 2 — Run Argon2id

Your Master Password (as UTF-8 bytes) is the Argon2id password input. The salt from Step 1 is the salt input. The function runs with these parameters:

m=… kB  ·  t=…  ·  p=…  ·  dkLen=…

Argon2id fills 64 MiB of memory during hashing, making brute-force attacks expensive even with specialised hardware.

Step 3 — Encode to a password string

The 32-byte output is Base64-encoded, + and / replaced with - and _ for compatibility, padding stripped, and the result trimmed to 16 characters. That is the password shown in the result box.

The master password is cleared from the input field immediately after it is read — before hashing begins. Nothing is ever sent to a server or written to storage. The full 32-byte hash and the salt are shown in the details section so you can verify or migrate the result independently.

🔬 How to verify independently

You can reproduce the exact same result using any standard SHA-256 tool and any standard Argon2id tool — without this app. This confirms the implementation is correct and gives you a recovery path if the app is ever unavailable.

Step 1 — Reproduce the salt
  1. Go to an online SHA-256 tool, for example emn178.github.io/online-tools/sha256.html.
  2. Enter the combined string: username:domain:pepper (use exactly the values you entered; username and domain are lowercased, pepper as-is, keep the trailing colon if pepper is empty: username:domain:).
  3. Make sure the tool hashes the UTF-8 text input (not a file) and outputs hex.
  4. Take the first 32 hex characters of the result (= 16 bytes). This is your salt. It should match the Salt (Hex) shown above.
Step 2 — Run Argon2id
  1. Go to an online Argon2 tool, for example www.devglan.com/online-tools/argon2-hash-generator.
  2. Set the parameters: ….
    3. p=1, hashLen=32.
  3. Enter your master password as the password.
  4. Enter the 32 hex chars from Step 1 as the salt (select hex input mode if available, otherwise convert to Base64 first — the Salt (Base64) value shown above is ready to paste).
  5. Run the hash. The raw output bytes, Base64-encoded, should match the Full Hash (Base64) shown above.
Step 3 — Reproduce the password string
  1. Take the Base64 output from Step 2.
  2. Replace all + with - and all / with _. Remove any trailing = padding.
  3. Keep the first 16 characters. This is your password.
Tip: the Salt (Hex) and Full Hash (Base64) in the details section are the checkpoints — if Step 1 matches Salt (Hex) and Step 2 matches Full Hash (Base64), your independent run is identical to what this app produced.
